## 移动端适配相关问题

### 1px问题
是由于 高清屏（Retina）也就是DPR = 2/3上，css的1px会被渲染成2px/3px物理像素，显得很粗
解决方案1 ：伪元素 + transform
```javascript
.border-1px {
  position: relative;
}

.border-1px::after {
  content: '';
  position: absolute;
  left: 0;
  top: 0;
  width: 200%;
  height: 200%;
  border: 1px solid #e5e5e5;
  transform: scale(0.5);
  transform-origin: 0 0;
  box-sizing: border-box;
  pointer-events: none;
}

/* DPR=3的设备 */
@media (-webkit-min-device-pixel-ratio: 3) {
  .border-1px::after {
    width: 300%;
    height: 300%;
    transform: scale(0.333);
  }
}

```

解决方案2 ：Viewport 设置缩放
解决方案3 ：box-shawdow 模拟一个
解决方案4 ：SVG背景（最精确）
解决方案5 ：PostCSS插件（工程化）


## 关于rem / vw 的使用

1. rem 适配
 
是以根元素的字体大小为基准，所有尺寸用这个rem相对单位

2. vw 适配
是以视口宽度为基准，1vw 就是视口宽度的 1%，纯 CSS 实现。


3. 一般字体不建议用vw 
因为屏幕越小的话，字就越小，可能会看不清楚。
